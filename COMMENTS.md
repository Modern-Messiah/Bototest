# Комментарии к выполненному заданию

## Плюсы реализации
- Чистая архитектура с разделением на слои routes, service, database
- Использование стандартной библиотеки sqlite3 без ORM
- Полное покрытие тестами основной функциональности
- Конфигурация через переменные окружения

## Минусы / Что не готово
- Нет rate limiting для защиты от спама
- Нет аналитики переходов по ссылкам
- Нет валидации на дублирование одинаковых URL

## Как бы развивал проект
- Добавил бы Redis для кэширования частых редиректов
- Реализовал бы статистику кликов по каждой ссылке
- Добавил бы опциональный TTL для ссылок
- Миграции базы данных через Alembic
